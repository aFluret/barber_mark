# Техническое задание (MVP)

## 1. Назначение
Разработать Telegram-бота для барбершопа, который позволяет клиенту записаться самостоятельно за 10 секунд, а барберу — получать актуальные списки записей.

## 2. Цели MVP
- Минимизировать количество шагов в пользовательском сценарии.
- Исключить ошибки выбора времени за счет работы только с готовыми слотами.
- Обеспечить автоматические напоминания перед визитом.

## 3. Роли и сценарии

### 3.1 Клиент
1. Запускает бота через `/start`.
2. Один раз передает контакт через кнопку Telegram и вводит имя.
3. Выбирает действие `📅 Записаться`.
4. Выбирает дату и свободный слот времени.
5. Подтверждает запись.
6. Получает сообщение: `Вы записаны на [дата] в [время]`.

Кнопки меню клиента:
- `📅 Записаться`
- `📋 Моя запись`
- `❌ Отменить запись`

Уведомления:
- Подтверждение после успешной записи.
- Напоминание за 24 часа.
- Напоминание за 2 часа (обязательно).

### 3.2 Барбер (администратор)
Команды:
- `/today` — записи на сегодня.
- `/tomorrow` — записи на завтра.
- `/all` — все будущие записи.
- `/schedule` — показать текущий рабочий график барбера.
- `/set_schedule ...` — сохранить график (однократно для MVP).

Формат вывода:
- `10:00 — Иван (+375291234567)`
- `11:00 — Артём (+375297654321)`

Дополнительно:
- Одноразовая настройка графика (рабочие дни и рабочие часы).
- Автогенерация слотов по графику.

Формат `/set_schedule`:
- `/set_schedule 1,2,3,4,5 10:00 18:00`
- где `1=Пн ... 7=Вс`, `10:00` — начало, `18:00` — конец.

## 4. Функциональные требования

### 4.1 Регистрация пользователя
- Сохранять `phone` только после явного нажатия кнопки `Поделиться контактом`.
- Сохранять имя пользователя после ввода.

### 4.2 Запись
- Длительность слота: 60 минут (фиксированный шаг в MVP).
- Занятые слоты не отображаются в интерфейсе выбора.
- При подтверждении создается запись со статусом `confirmed`.

### 4.3 Отмена
- При отмене статус записи становится `cancelled`.
- Слот освобождается немедленно и становится доступен для новых записей.

### 4.4 Админ-просмотр
- Команды возвращают отсортированный список записей по времени.
- Показываются имя и телефон клиента.

### 4.5 Напоминания
- Напоминания планируются сразу после подтверждения записи.
- Отправка производится за 24 часа и за 2 часа до визита.

## 5. Нефункциональные требования
- Простота UX: приоритет выше расширенной функциональности.
- Время прохождения сценария записи: до 10 секунд.
- Понятная архитектура и читаемый код для передачи другому разработчику.
- Отсутствие зависших слотов после отмен.

## 6. Ограничения MVP
Из MVP исключить:
- Сложные вложенные меню.
- Ручной ввод времени.
- Календарь с ручным выбором каждого дня.
- Дополнительные поля (комментарии, услуги).

## 7. Модель данных

### 7.1 Таблица `Users`
- `user_id` (Telegram ID)
- `phone`
- `name`
- `created_at`

### 7.2 Таблица `Appointments`
- `id`
- `user_id`
- `date`
- `time_slot`
- `status` (`confirmed` / `cancelled` / `completed`)
- `created_at`

## 8. Рекомендованный стек
- Python + aiogram 3.x
- Supabase (PostgreSQL) для хранения `users` и `appointments`
- APScheduler для напоминаний

Требуемые переменные окружения (бот-сервер):
- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY` (ключ с правами на запись; используется только на сервере)

## 9. Структура поставки
Ожидается:
1. Структура проекта (файлы и модули).
2. Основные хендлеры (`start`, запись, отмена, админ-команды).
3. Схема БД (SQL/ORM-описание).
4. Логика напоминаний.
5. Инструкция запуска (токен, переменные окружения).

## 10. Критерии приемки
- Запись клиента проверочно проходит за 10 секунд.
- После отмены слот сразу доступен.
- Напоминания приходят в заданные интервалы.
- Барбер видит актуальные записи через `/today`.
- Код и структура понятны для сопровождения.
