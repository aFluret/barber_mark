## [2026-03-23] - Реализована бизнес-логика записи (FSM + Supabase)

### Добавлено
- FSM сценарии регистрации и записи: `RegistrationStates`, `BookingStates`.
- Реальная регистрация через контакт Telegram + ввод имени.
- Выбор даты/времени и подтверждение записи (через inline-клавиатуры).
- Отмена активной записи клиента.
- Supabase-репозитории `users` и `appointments` (CRUD для MVP).
- Админ-команды `/today`, `/tomorrow`, `/all` с форматированным выводом (имя + телефон).

### Изменено
- Обновлены хендлеры `start`, `booking`, `appointment`, `admin` под реальный поток.

### Исправлено
- Убрана демо-логика выборки слотов, вместо нее используется выборка занятых слотов из БД.

## [2026-03-23] - Публикация в GitHub и подготовка схемы Supabase

## [2026-03-23] - Устойчивость к unique_violation при записи

### Добавлено
- Перехват ошибки уникальности при вставке `appointments` в `confirmed` слоте.
- Преобразование unique_violation в `SlotUnavailableError`, чтобы UI корректно сообщал “слот занят”.

### Изменено
- Убрана неиспользуемая сущность из репозитория (`SlotKey`).

## [2026-03-23] - Исправлено insert().select для supabase-py

### Добавлено
- После вставки appointments выполняется отдельный `select`, чтобы вернуть созданную запись.
- UI-обработчик подтверждения теперь показывает сообщение при неожиданных исключениях.

### Изменено
- Убрано использование `.select()` в цепочке после `.insert()` (не поддерживается supabase-py 2.x).

### Изменено
- В `ScheduleService` изменен шаг слота `DEFAULT_STEP_MINUTES` на `60`.
- В `src/bot/keyboards/booking.py` изменен layout inline-клавиатур: даты и слоты отображаются по 3 кнопки в ряд.
- В `docs/TZ_MVP.md` обновлено описание длительности слота до 60 минут.
