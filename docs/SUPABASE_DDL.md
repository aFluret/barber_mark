# SUPABASE DDL (TZ_MARK)

Документ предназначен для выполнения в SQL Editor Supabase (schema `public`).
Он описывает минимальную схему для interval-based записи:
- `services` — справочник услуг (цена и длительность)
- `appointments` — записи с `start_time/end_time` и защитой от пересечений для `status='confirmed'`
- `work_schedule` — график работы (дни недели + интервалы)
- `reminder_jobs` — запланированные напоминания для `ReminderService`

> Примечание: логика кода опирается на half-open интервалы `[start_time, end_time)`:
> граница `end_time == other.start_time` считается допустимой (не пересечение).

---

## 1) `users`

```sql
create table if not exists public.users (
  user_id bigint primary key,
  phone text not null,
  name text not null,
  role text not null default 'client',
  created_at timestamptz not null default now()
);
```

## 2) `services`

create table if not exists public.services (

## 4) `appointments` (interval-based)

```sql
create table if not exists public.appointments (
  id bigint generated by default as identity primary key,
  user_id bigint not null references public.users(user_id) on delete cascade,
  service_id integer not null references public.services(id) on delete restrict,

  date date not null,
  start_time time not null,
  end_time time not null,

  status text not null check (status in ('confirmed', 'cancelled', 'completed')),
  created_at timestamptz not null default now(),

  constraint appointments_start_end_check check (start_time < end_time)
);

create index if not exists appointments_date_status_idx on public.appointments(date, status);
```

### 4.1 Exclusion constraint (anti-overlap)

Запрещаем пересечения интервалов **только** для `status='confirmed'`.
Полуинтервал `[start, end)` обеспечивает корректную границу `end_time == other.start_time`.

```sql
-- Требуется:
-- - gist index для exclusion constraint создается автоматически
-- - tsrange строится как [date + start_time, date + end_time)

alter table public.appointments
  add constraint appointments_confirmed_no_overlap
  exclude using gist (
    tsrange(
      (date::timestamp + start_time),
      (date::timestamp + end_time),
      '[)'
    ) with &&
  )
  where (status = 'confirmed');
```

> Если constraint уже существует и вы переисполняете DDL повторно:
> удалите старый constraint перед добавлением.

## 5) `reminder_jobs`

```sql
create table if not exists public.reminder_jobs (
  id bigint generated by default as identity primary key,
  appointment_id bigint not null references public.appointments(id) on delete cascade,
  user_id bigint not null,

  remind_type text not null,
  remind_at timestamptz not null,
  sent_at timestamptz null,

  created_at timestamptz not null default now(),

  constraint reminder_jobs_unique_per_appointment_type
    unique (appointment_id, remind_type)
);

create index if not exists reminder_jobs_due_idx on public.reminder_jobs(remind_at);
```
